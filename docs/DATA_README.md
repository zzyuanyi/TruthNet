### 字段说明

| # | 字段名         | 类型 | 说明                                                 |

| 1 | `session_id` | int | 会话编号，共 35 个独立会话                           |

| 3 | `think_flag` | bool | 是否需要深度思考（True=77条，约 5.5%；False=1333条） |

### think_flag 含义分析

`think_flag` 是官方对问题难度的分层标记，区分了**简单查询**和**深度推理**两类问题：

#### think_flag=True（77 条，~5.5%）—— 🔬 深度推理题

需要**多步推理 + 多源数据整合**才能回答。这是拉开差距的关键题目，也是我们 4 Agent + 2 Skill 架构的主战场。

| 类型 | 典型问题 |
|------|------|
| 🔬 深度分析 | "嘉麟杰最新季度单季度净利润看点解读"、"华工科技的主力控盘度如何？" |
| 📊 技术研判 | "日出东方MACD分析"、"宗申动力的分时K线显示了什么信号？" |
| 💡 策略决策 | "双环传动浮盈17个点,该继续持有吗？"、"德瑞锂电当前适合加仓还是减仓？" |
| 🔍 复杂筛选 | "连续横盘30个交易日"、"哪些股近22天涨跌幅很小？" |
| 🏭 行业研究 | "有哪些企业在固态电池技术上有新产品？"、"人形机器人有哪些投资机会" |
| 🧠 元认知 | **"你这自选日报小助手只会找数据，不会说对策有什么意义"** |

> 元认知问题说明：评测方期望系统**不仅罗列数据，还要给出分析结论和可操作建议**。这与 V5 方案中「问答生成 Agent」的"预警点-数据对比-可能造假模式"三要素完全对应。

#### think_flag=False（1,333 条，~94.5%）—— 📋 简单查询题

单步查询、事实性问答，单一数据源即可回答，不体现系统差异化优势。

| 类型 | 典型问题 |
|------|------|
| 💰 价格查询 | "中国重工今天的股价是多少？"、"海通证券股价是多少？" |
| 📋 列表查询 | "证券板块的龙头股票是哪些"、"推买入哪些证券股票" |
| 🔁 日期循环 | 联得装备/金奥博连续十几天的"X月X日主力资金流向"（大量重复模式查询） |
| 📰 简单事实 | "中国重工是退市了？"、"AI行业近期有哪些新政策？" |

#### 评测策略建议

- **True 题（77条）**是评测核心——需要 4 Agent + 2 Skill 全线协作，体现我们的差异化优势
- **False 题（1333条）**考察基础能力——系统能回答但不会拉开差距，保证不丢分即可
- 深度题中多条涉及"对策/策略/投资建议"，**问答生成 Agent 不能只堆数据，必须有分析结论**
- **同一问题在不同轮次flag不同**：赛方按对话状态标注而非按句子——"最近怎么样"在第2轮=False，在第28轮（已聊27轮）时=True

#### 官方评分公式（2026-07-07赛方回信确认）

```
总分 = 0.5 × 多轮QA各项Answer准确性评估 + 0.5 × 最终复杂Query准确性评估
单项准确率 = 0.6 × 结果正确性 + 0.4 × 语义相似度
```

> 无标准答案，语义相似度占40%。策略：简单题保底不丢分，77道深度题（往往是会话末尾复杂Query）拉开差距。

### 用途

官方提供的问答测试集，用于评测系统的多轮对话能力和回答质量。对应 V5 方案中任务①（长对话记忆）和任务③（问答生成）的验收评测。77 条深度题为 Phase 4 评测的核心目标。

## 开发注意事项

1. **数据不出本地** — 数据集通过保密承诺书获取，仅用于比赛开发
2. **xlsx 用 pandas 读取** — `pd.read_excel()` 需安装 `openpyxl`
3. **CSV 编码 UTF-8** — 所有 CSV 统一 UTF-8，无需额外处理
4. **数值字段为字符串** — CSV 中数值以字符串存储，入库时需转换为 DECIMAL
5. **statement_type** — 三表全部为 `408006000`（母公司报表），无合并报表。分析时注意母公司口径限制
6. **Wind 代码为主键** — `s_info_windcode` 是文件夹 2/3/4 跨表关联的核心键
7. ⚠️ **研报 `sec_code` 无后缀** — 文件夹 5 的 `sec_code` 格式为纯数字（如 `601033`），与其他文件夹的 `s_info_windcode`（如 `600519.SH`）不同。跨表关联需通过 `exchange_code` 补后缀：`XSHG`→`.SH`，`XSHE`→`.SZ`
8. **文件夹 3 的 dict 文件名为 `ditct.txt`**（官方原始拼写，非错误）
9. ⚠️ **行情数据缺失** — 官方数据集描述提及"季度末复权收盘价、流通市值"，但实际未提供。MVP 阶段不答行情类问题，后续可接 akshare API 补充
10. ⚠️ **非结构化文本缺失** — 官方描述提及"附注、审计报告、监管问询函"，实际未提供。公告 PDF 链接（文件夹 3 `n_info_annlink`）可爬取补充
