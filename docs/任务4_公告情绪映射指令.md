# Phase B · 任务4：公告情绪映射（33类 fcode → 正/负/中性）

> **负责人**：数据组 | **截止**：2026-07-26 | **依赖**：无
> **交付物**：`announcements.sentiment` 字段（MySQL 表），29 类基础 fcode + 多 fcode 组合全覆盖
> **验收标准**：全部 7,311 条公告记录均有 `sentiment` 值（positive / negative / neutral）

---

## 0. 前置说明

### 0.1 数据概况

| 项 | 值 |
|---|---|
| 数据文件 | `赛题数据/3/clean.xlsx` |
| 总记录数 | 7,311 |
| 覆盖股票 | 2,585 只 |
| fcode 字典定义的基础类型 | 29 类 |
| 实际出现的 fcode 组合 | 142 种（多 fcode 用 `\|` 分隔） |

### 0.2 核心挑战

- 每条公告的 `n_info_fcode` 可能包含**多个**类型代码（用 `|` 连接），如 `5507060000|5506190000` 表示"违纪违规 + 个股其他公告"
- 训练数据集中**没有**人工标注的情绪标签，因此 Phase B 采用**规则映射**而非机器学习
- 部分 fcode 本身不携带情绪方向（如"股东大会""公司资料变更"），需中性处理

### 0.3 设计原则

1. **确定性优先**：每条公告的情绪必须由 fcode 规则计算得出，不依赖 LLM
2. **负面优先**：多 fcode 组合中，只要有一条为负面，整体判为负面
3. **可复现**：规则映射表版本化，情绪判定结果可重新生成
4. **有溯源**：每条记录保存 `sentiment_method` 字段，标明判定方式

### 2.2 多 fcode 组合规则

```text
对于带有 '|' 分隔符的多 fcode 公告（如 "5507060000|5506190000"）：

规则 1（负面优先）: 
  拆分所有 fcode → 逐个查表 → 若有任一 fcode 为 negative → 整体 negative

规则 2（正面次之）:
  若无 negative，但有 positive → 整体 positive

规则 3（中性兜底）:
  全为 neutral → 整体 neutral

示例:
  "5507060000|5506190000" = negative(违纪违规) + neutral(其他) → negative ✅
  "5506050000|5506010000" = positive(重大合同) + neutral(股东大会) → positive
  "5507200000|5507270000" = neutral(增减持) + neutral(人事) → neutral
```

import pandas as pd
import json
from collections import Counter

DATA = "赛题数据-14-知识图谱与智能推荐赛道-东吴证券-基于 Agentic AI 的金融长上下文推理、图谱穿透与财报反欺诈智能问答算法探索/3/clean.xlsx"
df = pd.read_excel(DATA)

# ── 情绪映射表 ─────────────────────────────────
FCODE_SENTIMENT = {
    "5107000000": "positive",   # 利润分配
    "5203000000": "negative",   # 质押冻结
    "5219000000": "positive",   # 回购股权
    "5230000000": "neutral",    # 权益变动
    "5404000000": "neutral",    # 补充更正
    "5406000000": "neutral",    # 业绩预告 (Phase C细化)
    "5502010000": "negative",   # 特别处理
    "5502040000": "negative",   # 终止上市
    "5506010000": "neutral",    # 股东大会
    "5506040000": "negative",   # 风险提示
    "5506050000": "positive",   # 重大合同
    "5506100000": "negative",   # 澄清公告
    "5506140000": "neutral",    # 停牌提示
    "5506160000": "neutral",    # 中介公告
    "5506170000": "negative",   # 法律纠纷
    "5506180000": "neutral",    # 公司资料变更
    "5506190000": "neutral",    # 个股其他公告
    "5506200000": "neutral",    # 其他补充更正
    "5506220000": "positive",   # 员工持股
    "5507040000": "neutral",    # 关联交易
    "5507060000": "negative",   # 违纪违规
    "5507200000": "neutral",    # 股份增减持
    "5507210000": "neutral",    # 资金投向
    "5507220000": "neutral",    # 资产重组
    "5507230000": "neutral",    # 收购兼并
    "5507240000": "negative",   # 借贷担保
    "5507260000": "neutral",    # 政策影响
    "5507270000": "neutral",    # 人事变动
    "5508000000": "negative",   # 函件
}

def classify_sentiment(fcode_str):
    """根据 fcode（可能含多类型）判定情绪。"""
    if pd.isna(fcode_str) or str(fcode_str).strip() == '':
        return "neutral", "fallback_empty"
    
    codes = str(fcode_str).split('|')
    sentiments = []
    unknown_codes = []
    
    for code in codes:
        code = code.strip()
        if code in FCODE_SENTIMENT:
            sentiments.append(FCODE_SENTIMENT[code])
        else:
            sentiments.append("neutral")
            unknown_codes.append(code)
    
    # 负面优先
    if "negative" in sentiments:
        method = "rule_multi_negative_first"
    elif "positive" in sentiments:
        method = "rule_multi_positive"
    else:
        method = "rule_multi_neutral"
    
    if unknown_codes:
        method += f"_unknown_{len(unknown_codes)}"
    
    # 返回最终情绪
    if "negative" in sentiments:
        return "negative", method
    elif "positive" in sentiments:
        return "positive", method
    return "neutral", method

# ── 执行标注 ─────────────────────────────────
results = []
for idx, row in df.iterrows():
    sentiment, method = classify_sentiment(row['n_info_fcode'])
    results.append({
        'object_id': row['object_id'],
        'wind_code': row['s_info_windcode'],
        'ann_dt': str(row['ann_dt']),
        'title': str(row['n_info_title']),
        'fcode_raw': str(row['n_info_fcode']),
        'sentiment': sentiment,
        'sentiment_method': method,
    })

result_df = pd.DataFrame(results)

# ── 统计 ─────────────────────────────────────
print("=== 情绪分布 ===")
dist = result_df['sentiment'].value_counts()
for s, c in dist.items():

print("\n=== 判定方法分布 ===")
for m, c in result_df['sentiment_method'].value_counts().head(10).items():
    print(f"  {m}: {c:>6,}")

print("\n=== 负面公告样本（前10条）===")
neg = result_df[result_df['sentiment'] == 'negative']
for _, r in neg.head(10).iterrows():
    print(f"  [{r['wind_code']}] {r['title'][:80]}")

# 验证：所有记录均有情绪值
null_sentiment = result_df['sentiment'].isna().sum()
print(f"\n=== 验收: 空值情绪 = {null_sentiment} {'✅' if null_sentiment==0 else '❌'} ===")

# ── 保存结果 ─────────────────────────────────
import os

        'version': '1.0.0',

    }, f, ensure_ascii=False, indent=2)
print("情绪映射表已保存: .local/generated/fcode_sentiment_map.json")

## 5. 边界情况处理

### 5.1 空 fcode 或缺失值
- 若 `n_info_fcode` 为 NULL → sentiment = `neutral`，method = `fallback_empty`
- 记录到质量标记中

### 5.2 不在字典中的 fcode
- 若出现 29 类之外的 fcode（如旧版代码） → 标记为 `neutral`，method 中附加 `_unknown_N`
- 实际出现此类情况时记录警告

### 5.3 多 fcode 中正负同时存在
- 按"负面优先"原则判定为 negative
- 这是合理的设计：一份公告同时涉及"重大合同"和"违纪违规"，应按更严重的信号（负面）处理

### 5.4 Phase C 升级路径
- 业绩预告（5406000000）：可解析标题中的具体预告方向（预增/预减/预亏）
- 股份增减持（5507200000）：可根据标题关键词区分增持（positive）和减持（negative）
- 资产重组（5507220000）/ 收购兼并（5507230000）：可结合具体内容判断

```sql

ADD COLUMN sentiment VARCHAR(10) NOT NULL DEFAULT 'neutral' 
    COMMENT 'positive|negative|neutral',

    COMMENT 'rule_multi_negative_first|rule_multi_positive|rule_multi_neutral|fallback_empty';
```
